AWS Cloud Computing Study Guide

This comprehensive study guide is designed to help you master AWS Cloud Computing from fundamentals to advanced services. Use this guide alongside your course materials to enhance your learning experience.

1. Introduction to AWS

What is AWS?

Amazon Web Services (AWS) is a comprehensive cloud computing platform provided by Amazon. It offers a mixture of infrastructure as a service (IaaS), platform as a service (PaaS), and packaged software as a service (SaaS) offerings.

Why Use AWS?

  • Flexibility - Only use what you need, scale up or down as required
  • Cost-Effective - Pay only for what you use, no upfront hardware costs
  • Scalability - Easily scale resources up or down based on demand
  • Security - Comprehensive security capabilities and features
  • Global Reach - Deploy applications worldwide in minutes

AWS Account Structure

  • Root Account - The initial account created when first setting up AWS
  • IAM Users - Individual users created within the account
  • Organizations - Manage multiple AWS accounts as a single unit
  • Consolidated Billing - Single payment method for all accounts

2. AWS Global Infrastructure

Regions

  • Geographical areas with multiple Availability Zones
  • Each region is completely independent
  • Not all services are available in all regions
  • Data does not automatically replicate across regions

Availability Zones

  • Physically separate data centers within a region
  • Connected with high-bandwidth, low-latency networking
  • Designed for fault isolation
  • Applications should use multiple AZs for high availability

Edge Locations

  • Content Delivery Network (CloudFront) endpoints
  • More numerous than regions or AZs
  • Used to cache content closer to users
  • Also used for Route 53 (DNS) and AWS Shield

3. Core AWS Services

Identity and Access Management (IAM)

IAM enables you to manage access to AWS services and resources securely. You can create and manage AWS users and groups, and use permissions to allow and deny their access to AWS resources.

  • Users, Groups, Roles, and Policies
  • Multi-Factor Authentication (MFA)
  • Temporary security credentials
  • Identity federation

Virtual Private Cloud (VPC)

Amazon VPC lets you provision a logically isolated section of the AWS Cloud where you can launch AWS resources in a virtual network that you define.

  • Complete control over virtual networking environment
  • Subnets, Route Tables, Internet Gateways
  • Security Groups and Network ACLs
  • VPN and Direct Connect

4. Compute Services

EC2 (Elastic Compute Cloud)

Virtual servers in the cloud that provide resizable compute capacity.

  • Various instance types for different use cases
  • Multiple pricing options (On-Demand, Reserved, Spot)
  • Auto Scaling to adjust capacity
  • Load Balancing to distribute traffic

Lambda

Run code without provisioning or managing servers.

  • Serverless compute service
  • Pay only for compute time consumed
  • Automatically scales applications
  • Supports multiple programming languages

Elastic Beanstalk

Easy-to-use service for deploying and scaling web applications.

  • Platform as a Service (PaaS)
  • Automatic capacity provisioning
  • Load balancing and auto-scaling
  • Supports various platforms (Java, .NET, PHP, etc.)

5. Storage Services

S3 (Simple Storage Service)

Object storage service that offers industry-leading scalability, data availability, security, and performance.

  • Virtually unlimited storage capacity
  • Different storage classes for different use cases
  • Versioning, encryption, and access control
  • 99.999999999% durability

EBS (Elastic Block Store)

Persistent block storage volumes for use with EC2 instances.

  • Independent lifecycle from EC2 instances
  • Different volume types for different performance needs
  • Point-in-time snapshots
  • Encryption available

Glacier

Low-cost storage service for data archiving and long-term backup.

  • Designed for infrequently accessed data
  • Various retrieval options (minutes to hours)
  • Highly durable storage
  • Significantly lower cost than S3

6. Database Services

RDS (Relational Database Service)

Managed relational database service that makes it easy to set up, operate, and scale a relational database.

  • Supports multiple database engines
  • Automated backups and patching
  • Multi-AZ deployments for high availability
  • Read replicas for improved performance

DynamoDB

Fast and flexible NoSQL database service for any scale.

  • Fully managed NoSQL database
  • Single-digit millisecond performance
  • Automatic scaling of throughput capacity
  • Built-in security and backup/restore

Redshift

Fast, fully managed data warehouse that makes it simple and cost-effective to analyze data.

  • Petabyte-scale data warehouse service
  • Columnar storage technology
  • Massively parallel processing (MPP)
  • SQL interface for analytics

7. Networking & Content Delivery

Route 53

Highly available and scalable cloud Domain Name System (DNS) web service.

  • Domain registration
  • DNS routing policies
  • Health checking and monitoring
  • Traffic flow and latency-based routing

CloudFront

Fast content delivery network (CDN) service that securely delivers data, videos, applications, and APIs.

  • Global network of edge locations
  • Low latency and high transfer speeds
  • Integration with AWS Shield for DDoS protection
  • Customizable content delivery

8. Security & Compliance

AWS Shield

Managed Distributed Denial of Service (DDoS) protection service.

  • Always-on detection and automatic mitigations
  • Protects against most common DDoS attacks
  • No additional charge for AWS Shield Standard
  • AWS Shield Advanced for enhanced protection

AWS WAF

Web Application Firewall that helps protect web applications from common web exploits.

  • Protects against SQL injection and XSS
  • Create custom rules to block common attack patterns
  • Real-time metrics and sampled web requests
  • Integration with CloudFront and Application Load Balancer
